# الخطة التنفيذية النهائية — قسم البيان في مشكاة

**التاريخ:** 2026-07-18 · **الغرض:** خطة يقرؤها المالك فيقول «أطلق» — مبنية على
DESIGN/SOURCES/PROPOSAL وملحق المراجعة في `BAYAN-SESSION-PROMPT` الثاني، وعلى
**جرد متحقق بالعدّ الحتمي** من الحزمة المحلية (OpenITI RELEASE 2025.1.9 التي
نزّلها المالك). القواعد الحاكمة: ميثاق العبارة فوق الجميع؛ عينات مجمدة وعتبات
معلنة قبل أول دفعة؛ دفتر سرب قبل كل إطلاق؛ سونيت للأسراب؛ أجزاء مقيَّمة؛
النتائج تنشر نجاحًا أو إخفاقًا.

**ما تغيّر منذ PROPOSAL:** (١) المكتبة صارت محلية بالكامل — ١٨ شاهدًا لأحد عشر
كتابًا في `js/data/bayan-sources/raw/` ببصمات وأعداد متحققة (MANIFEST.md)؛
(٢) قرار المصادر المفتوحة نافذ (لا كشط من موقع الشاملة؛ OpenITI معتمد)؛
(٣) مبدأ نبراس الحاكم مسجل (§ج): الاستنباط قدرة مضافة لا هوية.

## المرحلة ب٠ — الهيكلة الحتمية للكتب (بلا أسراب، عمل قيادي)

تحويل كل كتاب من mARkdown إلى JSONL مداخل **على عقد البيانات الموحد «جاهز
لنبراس»** (`findings/NIBRAS-DATA-CONTRACT.md` — فيصير إدماج البيان في نبراس
لاحقًا عمل يومٍ لا مشروعًا):

```json
{
  "id": "bayan-furuq-0412",
  "layer": "bayan-furuq",
  "kind": "farq | wujuh | mawdic-mutashabih | naw-culum | qitca",
  "text": "…نص المدخل كاملًا…",
  "source": {"work": "الفروق اللغوية", "author": "أبو هلال العسكري",
             "locus": "الفرق بين الخوف والخشية",
             "witness": "…FuruqLughawiyya.JK006960-ara1"},
  "grade": "manqul",
  "anchor": {"term": "خوف/خشية", "root": ["خوف", "خشي"], "aya": ["13:21"]}
}
```

(خرائط الاستعمال والبطاقات تخرج بالعقد نفسه بدرجة `mahsub`؛ واللفتات — إن
أُقرت ق٧ — بدرجة `muwallad` بمقدماتها.)

- **المدخلية الخمسة** (فروق العسكري ١٠٣٠ · بصائر ٩٣١+١١١ · وجوه العسكري ٢٨٥ ·
  نزهة ٣٢٤ · الدامغاني ~٤٥٠): تقطيع آلي على علامات المداخل المتحققة، مع فحص
  اكتمال (مجموع القطع = طول النص؛ عيّنة يدوية ٢٠ مدخلًا لكل كتاب).
- **الموضعية** (درة التنزيل ١٠٢ · ملاك التأويل ٣١٣): تقطيع على العناوين
  المرقمة/`### |`؛ و`quran_refs` تُستخرج **حتميًّا** بمطابقة الاقتباسات القرآنية
  داخل المدخل على نص المصحف (أداة مطابقة بالهيكل المجرد — عندنا الخبرة من
  فروق التنزيل).
- **البابية** (البرهان ~٤٧ نوعًا · الإتقان ٧٥ معنونًا): تقطيع على الأنواع طبقةَ
  تصفحٍ واستشهادٍ بابيّ — **لا تدخل الفهرسة المصطلحية** في هذه المرحلة.
- **النظرية** (دلائل الإعجاز، أسرار البلاغة): تُحفظ خامًا لمرحلة البلاغة.

**الكلفة:** يوم إلى يومي عمل قيادي، صفر وكلاء. **بوابتها:** فحوص الاكتمال +
عينات الصحة العشرون لكل كتاب خضراء.

## المرحلة ب١ — الإسناد إلى جذورنا (حتمي أولًا، ثم سرب صغير بإذن)

الغاية: كل مدخل معجمي يُسند إلى جذور/لمّات QAC فيصير قابلًا للاستدعاء من
البطاقات ومن نبراس ومن بطاقة الكلمة.

1. **إسناد حتمي أول** بمطابقة الهيكل المجرد لرؤوس المداخل على جذور القاعدة
   (كما في `usage_map.py`): المتوقع حسم ٦٠–٧٠٪ آليًّا (رؤوس بصائر والدامغاني
   ونزهة جذرية الطابع أصلًا).
2. **سرب الإسناد** للبقية الغامضة (~١٠٠٠–١٢٠٠ مدخل): الوكيل يقرأ المدخل ويسند
   (جذر/جذور القرآن للطرفين + أطرافه القرآنية + وسم «غير قرآني» عند اللزوم).
   دفعات ١٠ مداخل → **~١٠٠–١٢٠ دفعة سونيت + شاهد عدل ثانٍ ٥٪** —
   **سقف معلن: ٢٠٠ دفعة** لا يُتجاوز إلا بقرار جديد.
3. **بوابات النزاهة (قبل أول دفعة):** عينة ضبط ٥٠ مدخلًا تُسند يدويًّا وتجمَّد
   بقفل commit + عينة مصونة ٢٥؛ العتبتان: دقة ≥٩٥٪ واستعادة ≥٩٥٪ على
   المصونة؛ دفتر سرب في SWARMS-REGISTRY؛ rubric بلغة الميثاق من مسودته الأولى.

**غلة المرحلة:** فهرس مسند يجعل لكل زوج قرآني قراءاته المنقولة جاهزة، ويولد
**قائمة انتظار بطاقات مرتبة بالدليل** (أزواج العسكري القرآنية أولًا).

الخمس المكتملة: **أتى/جاء · خوف/خشية · اسطاعوا/استطاعوا · القاسطون/المقسطون ·

| ٩ | بخل / شح | فرق متقاربين |
| ١٠ | نعمة / نعيم | مشترك جذري مفترق |
| ١١ | رأفة / رحمة | فرق متقاربين |
| ١٢ | عهد / ميثاق | فرق متقاربين |
| ١٣ | اصطفى / اجتبى / اختار | فرق ثلاثي |
| ١٤ | ظن / حسب / زعم | فرق ثلاثي |
| ١٥ | سقى / أسقى | صيغتان من جذر |

للبطاقة ≈ يومان مركّزان. **بوابتها:** مراجعة المالك عينةً (٣ بطاقات) قبل
إتمام العشرين.

## المرحلة ب٣ — واجهة القسم (البند ٥: تصور؛ التنفيذ قرار منتج مستقل)

```
/bayan  ─ رأس: «البيان — تدبر لغة القرآن الخاصة» + بحث ميسّر (كلمة/زوج)
  ├─ شبكة البطاقات مصنفةً بأنواعها الخمسة (شارة لكل نوع + عدّاد مواضع)
  ├─ بطاقة المصطلح (عقد العرض الثابت):
  │    رأس + سطر الكشف المحسوب
  │    ← لوحات تدريجية (أزمنة/مصاحبات/مواضع بوسم وحدة السياق)
  │    ← «قراءات» مطوية (المنقول المسند من الفهرس، بمصادره)
  │    ← «منهج البطاقة» مطوي (الاستعلامات وإعادة التشغيل)
  ├─ «مكتبة الباب»: تصفح الكتب المهيكلة نفسها (مدخلًا مدخلًا، بحثًا ورصًّا)
  │    — طبقة التنظيم تخدم القارئ مباشرة لا البطاقات فقط
  └─ ربط: بطاقة الكلمة في القارئ ← «هذه الكلمة في باب البيان» عند التغطية
```

لا نشر ولا مساس بالتطبيق قبل إذن المالك؛ البناء الأنسب بعد ت١–ت٣ أو بالتوازي
دون نشر (لا يزاحم الأسراب).

## المرحلة ب٤ — طبقة الاستكشاف (البند ٣: هيكل وقرار، لا كلفة ملزمة)

**القرار الأسبق (للمالك، مستقل معنون):** توليد اللفتات يعارض حدّ DESIGN §٤
(«الحساب يصف ولا يعلّل») — المطلوب إقرار **تعديل الحد**: يُسمح بتعليلٍ
**مولَّد موسوم** («فرضية مولَّدة قيد الفحص — ليست من كلام العلماء») خاضعٍ
لفحص عدائي، مع القيد المضموني: العلة تفسر النمط التوزيعي المحسوب ولا تتكلم
في حكمة الاختيار الإلهي (يُضاف سطرًا في TERMS-CHARTER).

**الهيكل عند الإقرار:** rubric يجسد منهج الفروق (خريطة محسوبة ← فرضية علة ←
شاهد ومضاد) بلغة الميثاق؛ وأضلع الفحص الخمسة المقررة في الملحق: عينة إعادة
اكتشاف مصونة + عينة ضد (مقارنات واهية يُمتحن الإحجام عنها) + **اختبار تضليل
بخرائط معكوسة** (كاشف الاعتماد على الذاكرة) + مطابقة آلية لأرقام اللفتة على
الخريطة + عينة أزواج لا قراءة منشورة لها بتحكيم بشري بمعيار مكتوب. تجربة
أولية (dry-run) على مجموعة منفصلة قبل تجميد المصونة؛ الدفعة التجريبية
المقترحة: **٢٠ زوجًا** ببوابة كاملة قبل أي توسيع.

**أسئلة مفتوحة تُحسم في جلسة تصميمها المستقلة:** تركيب العينتين وأحجامهما
النهائية على الأنواع الخمسة؛ عتبتا النجاح الرقميتان؛ صيغة العرض للقارئ.
إن أخفق الامتحان: يُنشر الإخفاق ويبقى القسم تنظيميًّا — **الخسارة محدودة
بالتصميم** (الفهرس والبطاقات قائمة بذاتها).

## المرحلة ب٥ — نبراس المستنبط (البند ٦: معمار بمبدأ §ج، بعد اكتمال القسم)

- **معمار قدرات لا هوية**: أداة استنباط جديدة تنضم إلى عُدّة نبراس القائمة
  (سياق، بحث، تفاسير، كتب) يوجَّه إليها السؤال البياني فقط؛ استرجاعها من
  خرائط الاستعمال + الفهرس المسند؛ حارس المقدمات (امتداد
  enforceVerbatim/unbackedQuote إلى المقدمات) يسري على أجوبتها وحدها.
- **الوسم:** «استنباط مولَّد بمقدماته» — كل مقدمة مسندة إلى خريطة أو مدخل
  كتاب، والنتيجة قابلة للحفظ والمشاركة بمقدماتها.
- **طقم المسابر قبل الإتاحة:** سؤال بجواب معلوم (كـ﴿فأسر بعبادي ليلا﴾) ·
  سؤال لا جواب جيد له (يُمتحن إقرار الحد) · سؤال فخ بمقدمة غير موجودة
  (يُمتحن الامتناع) · **سؤال خارج نطاق الباب** (يُمتحن التحويل للمسار العام)
  · **مسابر عدم النكوص** من طقم v3 (تفسير وشرح وربط عامة).
- الكلفة والتفصيل بعد اكتمال ب٠–ب٢ (يلزمها تضمين متجهات الفهرس — تقدير أولي
  غير ملزم: ~٣٬٥٠٠ مدخل مهيكل للتضمين).
- **يتبعها بند المالك الجديد:** خطة «تغطية نبراس الشاملة» لكل أقسام مشكاة
  (MASTER-PLAN، توجيهات القسم، البند ٤).

| # | القرار | التوصية |

| ق٥ | **ابن القيم**: مدارج السالكين (+ بدائع الفوائد وأمثال القرآن) متاحة محليًّا | التوصية: تنضم للفئة (ب) في مرحلة تالية لا الآن |

**تنسيق المجريين (مثبت مع جلسة نبراس):** جلسة البيان جلسةُ بياناتٍ تسلّم
ملفاتٍ بالعقد الموحد ولا تمس كود المساعد؛ وبناء «نبراس الشامل» في جلسة نبراس
على معمار الأداتين العامتين (بحث بطبقة مسماة + استدعاء بمرسى) — القسم الجديد
قيمةُ layer لا عقدُ أداة. جرد الطبقات القائمة: `findings/NIBRAS-LAYERS-INVENTORY.md`.
وقاعدة نظافة مشتركة: **الإضافة إلى git بأسماء الملفات صراحةً** — لا `git add -A`
في مجلد العمل المشترك.

**تسلسل التنفيذ الموصى به عند الإقرار:**
ب٠ (يومان) ← بوابة ← ب١ (سرب صغير ببواباته) ← ب٢ (العشرون ببوابة الثلاث) ←
ب٣ الواجهة بقرار منتج ← ب٤ وب٥ بجلستي تصميم مستقلتين ← خطة تغطية نبراس
الشاملة. **لا يتزامن سربان ثقيلان**: أسراب البيان لا تُطلق أثناء موجات
التوحيد الجارية (ب١ صغير ويُجدول في فراغ النوافذ).
