# خطة تطبيقات مشكاة: إلكترون + جوال Expo + مفاتيح المستخدم الثلاثة

**التاريخ:** 2026-07-19 · **التوقيت:** بعد إتمام مشكاة الحالية (الثلاثية الحسابية
+ قسم البيان + النشر الموحد) · **المرجع الجوال:** `/Volumes/data/jadwal-v2/jadwal/mobile/tadween`
(Expo + expo-router + EAS + خط Tajawal + flash-list — بنية مجربة عند المالك).

## المبدأ الحاكم

**كلا التطبيقين يحمل كل بيانات مشكاة ويعمل بلا سيرفر إطلاقًا.** الشبكة تلزم
فقط لنداءات الذكاء الاصطناعي (بمفتاح المستخدم) ولجلب شظايا الكتب أول مرة
على الجوال (ثم تُخزَّن محليًّا فيعمل دون اتصال).

## ١ — إعادة هيكلة المونوريبو (الأساس المشترك)

```
js/
  packages/core-data/    ← يُستخرج من studio: محمّلات الطبقات (آيات، وحدات،
                            صلات، وجوه، فروق، متجهات int8، بحث المعنى والعنقود)
                            TS خالص + typed arrays — يعمل في الويب وRN وإلكترون
  apps/studio/           ← الويب كما هو (يستهلك core-data)
  apps/desktop/          ← إلكترون
  apps/mobile/           ← Expo
```

## ٢ — تطبيق إلكترون

- غلاف حول بناء studio نفسه + البيانات كلها في موارد التطبيق (عشرات م.ب +
  شظايا الكتب كاملة — الحجم غير مقيد على سطح المكتب).
- بدائل `api/*` محليًّا في العملية الرئيسية: نداءات المزود مباشرة بمفتاح
  المستخدم (لا Vercel).
- **السابقة الجاهزة**: مشروع الحديث يشحن مثبتات سطح مكتب فعلًا؛ والبناء عبر
  GitHub Actions على qataruts/quran (بند قائم في التذكيرات — بناء إلكترون
  محليًّا محدود في بيئة العمل).

## ٣ — تطبيق الجوال Expo

- على نمط tadween: expo-router، EAS للبناء، Tajawal، flash-list للقوائم،
  AsyncStorage/SecureStore.
- الواجهات تُبنى RN فوق core-data المشترك (لا يعاد بناء منطق البيانات) —
  البحث المتجهي int8 يعمل في Hermes بلا مشكلة لأحجامنا.
- البيانات: الطبقات الأساسية داخل الحزمة (~٣٠–٥٠ م.ب مقبولة للمتاجر)؛ شظايا
  الكتب تُجلب عند أول حاجة وتُخزَّن (offline-after-first-fetch) — نفس صيغة
  عقد التوافق في BOOKS-PIPELINE-PLAN.

## ٤ — مفاتيح المستخدم الثلاثة (قرار المالك: نبراس غير مقفل حتى يضيف المستخدم مفتاحه)

- **السياسة (تشمل الويب quran.uts.qa أيضًا):** نبراس يظهر للجميع، ويُفعَّل
  حين يضيف المستخدم مفتاحه: **Gemini أو OpenAI أو Anthropic**. المفاتيح تُخزن
  على جهاز المستخدم فقط (localStorage / SecureStore / safeStorage) ولا تمر
  بخوادمنا للتخزين أبدًا.
- **طبقة مزود موحدة** (chat adapter): gemini-2.5-flash/pro · GPT (الأحدث) ·
  Claude (الأحدث) — بنفس عقد الأدوات والاستنباط.
- **قيد تقني يُعرض بصدق في الواجهة:** بحث المعنى يتطلب أن يُضمَّن السؤال
  بنفس نموذج تضمين الفهرس (gemini-embedding-001) — فمفتاح Gemini (له طبقة
  مجانية) يفتح كل القدرات؛ ومفتاح OpenAI/Anthropic وحده يفتح محادثة نبراس
  وأدواته اللفظية والطبقات المحسوبة دون بحث المعنى. (خيار مستقبلي منفصل:
  نموذج تضمين محلي + إعادة تضمين الفهرس — قرار مستقل بكلفته.)
- **واقع CORS:** Gemini مباشر من المتصفح ✓ (مستعمل عندنا فعلًا)؛ Anthropic
  مباشر بترويسة `anthropic-dangerous-direct-browser-access` ✓؛ OpenAI لا يفتح
  CORS للمتصفح → على الويب يمر عبر `api/*` وسيطَ تمريرٍ لحظيًّا (المفتاح في
  الطلب، لا يُخزن)؛ وعلى إلكترون والجوال النداء مباشر بلا قيد.
