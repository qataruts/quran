# عقد البيانات الموحد «جاهز لنبراس» — يُلزَم به كل قسم

**التاريخ:** 2026-07-18 · **الأصل:** اقتراح جلسة نبراس (المصادَق عليه) بعد مبدأ
المالك «نبراس نافذة مشكاة الشاملة» · **الغرض:** أي قسمٍ يُخرج بياناته على هذا
العقد يصير إدماجه في نبراس عملَ يومٍ لا مشروعًا. **أول ملتزم به: قسم البيان**
(مرحلة الهيكلة ب٠ في `findings/bayan/FINAL-PLAN.md`).

## الصيغة الحاملة

ملفات `rag-<layer>.json` (+ `rag-<layer>.bin` متجهات int8 اختيارية) على السنة
القائمة في `js/apps/studio/public/` — **ملفات لا قواعد بيانات** (قرار مثبت في
أساس نبراس المتجهي). الثقيل يذهب إلى rag.db الخادمية بالبنية نفسها.

## قيد المدخلة الواحدة

```json
{
  "id": "bayan-furuq-0412",
  "layer": "bayan-furuq",
  "text": "نص المدخلة كاملًا…",
  "source": {
    "work": "الفروق اللغوية",
    "author": "أبو هلال العسكري",
    "locus": "الفرق بين الخوف والخشية",
    "witness": "OpenITI …FuruqLughawiyya.JK006960-ara1"
  },
  "grade": "manqul",
  "anchor": {
    "aya": ["13:21"],
    "root": ["خوف", "خشي"],
    "term": "خوف/خشية"
  }
}
```

- **`text`**: نص المدخلة الذي يُسترجع ويُقتبس.
- **`source`**: المصدر بموضعه الدقيق وشاهده الرقمي — إلزامي في المنقول،
  وفي المحسوب يشير إلى الطبقة والسكربت المولد.
- **`grade`** — درجات السند الثلاث (جوهر أمانة البوابة الشاملة):
  - **`manqul` منقول**: اقتباس حرفي منسوب — يحرسه حارس المطابقة الحرفية
    (سنة enforceVerbatim).
  - **`mahsub` محسوب**: أرقامنا وخرائطنا — تُنسب لطبقات مشكاة، ويحرسها فحص
    مطابقة الأرقام (المقر في مراجعة خطة البيان).
  - **`muwallad` مولَّد**: استنباط موسوم بمقدماته — يحرسه حارس المقدمات
    (امتداد unbackedQuote)، ولا يُعرض إلا بوسمه.
- **`anchor`**: مرسى الاستدعاء — آية (`s:a`) و/أو جذر/لمّة و/أو مصطلح؛ يجوز
  تعددها، ولا تخلو مدخلة من مرسًى.

## قواعد الإلزام

1. كل قسم جديد في مشكاة يسلّم بياناته لنبراس **بهذا العقد حصرًا**؛ وخطط
   الأقسام تحمل بند الإدماج القياسي: العقد + ثلاثة مسابر (معرفي، حدّي/فخ،
   وإعادة طقم عدم النكوص عبر test-assist.mjs).
2. سطح الأدوات **يعمَّم ولا يُكاثَر**: أداتا نبراس العامتان (بحث دلالي في
   طبقة مسماة `layer`، واستدعاء دقيق بمرجع/مرسًى) تقرآن هذا العقد — القسم
   الجديد قيمةُ `layer` جديدة + سطر دستور + مسابر، لا عقدَ أداةٍ جديدًا.
3. الإحالة جزء من العقد عمليًّا: لكل `layer` مسارُ شاشةٍ في مشكاة
   (`/read`، `/journey`، `/bayan/<card>`…) ليقود نبراس القارئ إلى موضعه.
4. جرد الطبقات القائمة ومواضع غيابها عن نبراس: `findings/NIBRAS-LAYERS-INVENTORY.md`.
